# Modules

本文档定义平台的模块划分与职责边界。

## 1. 数据面模块

负责入口流量的前置处理。

### 职责
- 接收镜像流量
- 基础识别、分类与特征提取
- 轻量规则匹配
- 选择响应执行路径
- 执行极少数同步 TX 响应动作，例如 TCP reset
- 将需要完整原包上下文的 TX 响应流量提交到 XSK
- 输出观测事件或动作结果

### 边界
- 不负责复杂编排
- 不负责深度分析
- 不实现复杂 XSK TX 响应动作
- 不依赖 ringbuf 传递构包所需原包字段
- 不负责页面展示
- 不负责响应策略决策

---

## 2. 控制面模块

负责平台的统一管理与流程编排。

### 职责
- 管理规则、策略和运行配置
- 校验响应动作语义
- 将规则响应编码为数据面 action
- 接收数据面输出
- 维护平台状态
- 编排分析、响应和展示流程

### 边界
- 不直接处理原始流量
- 不承担深度分析逻辑
- 不负责具体页面渲染
- 不负责底层动作执行

---

## 3. 深度分析接入模块

负责与后端分析系统对接。

### 职责
- 接收待分析的流量或事件
- 对接分析链路
- 统一分析输入与输出
- 回传分析结果

### 边界
- 不负责前置裁决
- 不负责规则管理
- 不负责主动响应执行
- 不承担后端分析系统内部逻辑

---

## 4. 响应执行模块

负责执行主动动作并反馈结果。

### 职责
- 接收响应指令
- 接收 XSK TX 响应原包
- 执行用户态 TX 响应动作
- 记录执行结果
- 提供基础失败反馈

### 边界
- 不负责规则判定
- 不负责 BPF 同步 TX 响应动作
- 不负责事件分析
- 不负责页面展示
- 不决定是否响应

---

## 5. 管理与展示模块

负责配置入口与运行结果展示。

### 职责
- 提供基础配置入口
- 展示平台状态
- 展示规则、事件和响应结果
- 提供基础查询能力

### 边界
- 不参与流量处理
- 不执行规则
- 不负责分析和响应编排
- 不承担核心链路逻辑

---

## 核心链路

镜像流量接入  
→ 数据面模块前置裁决  
→ TCP reset 由数据面同步 TX，其他 spoof 响应经 XSK 交给响应执行模块  
→ 控制面模块统一管理规则、状态与结果  
→ 按需进入深度分析接入模块  
→ 管理与展示模块提供配置与结果查看
